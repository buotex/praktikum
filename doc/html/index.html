libfbi: fast box intersection in C++

Introduction

libfbi is a header-only C++ template library that enables the efficient solution of box intersection problems in an arbitrary number of dimensions. The implementation is based on a publication by Zomorodian and Edelsbrunner (2000) and makes heavy use of C++ metaprogramming and variadic template programming techniques. Despite this complexity, the library provides a straightforward and simple interface that allows easy integration into new and existing projects.

libfbi has been developed to tackle feature extraction and data analysis tasks for high-resolution liquid chromatorgraphy/mass spectrometry (LC/MS) data sets. The library itself, however, is in no way limited to this application scenario: it can easily be used to approach many kinds of multi-dimensional box intersection and related problems (e.g. determining bounding volumes, k-nearest-neighbor search, density estimation, correspondence estimation in motion or pattern alignment tasks, etc.).

Citation

If you use libfbi in a publication, you must cite the following paper:

  • Kirchner M*, Xu B*, Steen JAJ, Steen H (2011). libfbi: A C++ Implementation for Fast Box Intersection and Application to Sparse Mass Spectrometry Data. Submitted.

libfbi development has been carried out in the Steen & Steen Lab: http://steenlab.org/

Requirements

Please note that libfbi makes heavy use of TR1 and C++0x features, in particular tuples and variadic templates. Development was carried out under GNU GCC 4.4, on an Ubuntu Linux system. Windows users currently need to make use of the MinGW compiler as variadic template support under Microsoft Visual Studio will only be available starting from Visual Studio 11.

libfbi itself is free of any third-party dependencies; the tests and examples make use of the VIGRA unittest header file and depend on the BOOST library. Memory tests require valgrind, coverage analysis depends on gcov.

Download

Tar archives with libfbi headers, a CMake config file (expecting the installation in /usr/local) and the complete HTML documentation are available from http://software.steenlab.org/libfbi

Installation

libfbi is a header-only library and does not require the installation of any compiled libraries. To use libfbi,

  • download a header file and documentation bundle,
  • include the header files in your project and
  • adjust the compilation flags of your compiler to support C++0x functionality (e.g. libfbi requires the -std=c++0x flag in gcc).

If you use the CMake build system, unpack the libfbi distribution to a convenient location and include the following lines into your CMakeLists.txt to use libfbi: 

[...]
find_package(LIBFBI)
if (LIBFBI_FOUND)
    include (${LIBFBI_USE_FILE})
    add_executable(MyProgram MyProgram.cpp)
endif(LIBFBI_FOUND)
[...]

If you include the share/libfbi/cmake directory of the libfbi distribution in the project search path, you can simply build your project using 

cd <my project build dir>
cmake [optional cmake flags for your project] <my project source dir>

If you do not include the directory, you need to tell find_package where to find the libfbi configuration information: 

cd <my project build dir>
cmake -DLIBFBI_DIR=<install_dir>/share/libfbi/cmake \
    [optional cmake flags for your project] <my project source dir>

Building from Source

The following illustrates how to build the examples, tests and distribution packages. Bundling libfbi requires a working CMake build system (available from http://cmake.org/) and CMake >= 2.6.

With cmake in the system path, the build process is 

 tar xvzf libfbi-xxxxxxx.tar.gz
 mkdir libfbi-build
 cd libfbi-build
 ccmake ../libfbi
 make
 make test

And, optionally: 

 make package
